# 1.静态库、动态库和第三方库

## 1.静态库

> windows下静态库文件名为`xlog.lib`(release版：二进制文件)或`xlog_d.lib`（debug版：字符文件）。Linux下静态库文件名为`libxlog.a` 或`libx.a` 。

静态库基本可以理解为编译后的二进制代码，类似.o



1. 准备程序

add.cpp
```cpp
#include "add.h"

int add(int a,int b){return a+b;}
```

add.h
```cpp
int add(int a,int b);
```

main.cpp
```cpp
#include<iostream>

#include "add.h"

using namespace std;

  

int main(){

    cout<<add(3,5)<<endl;

    return 0;

}
```

2. 生成二进制文件

```
//生成add.o二进制文件，而不是生成可执行文件
g++ -c add.cpp -o add.o
```

3. 创建静态库

在linux中静态库后缀名为`***.a`

```
//生成.a静态库，如果链接多个.o文件，直接在后面添加
ar crs libadd.a add.o
```

4. 使用静态库

```
//main.cpp：想要编译的程序文件
//-L 文件路径：找到静态库的路径，.表示当前路径
//-l库名称：使用库，库名称需要去掉前缀lib和后缀.a
//-I 可执行文件名称：生成可执行文件，文件名称自定
g++ main.cpp -L. -ladd -I main
```


## 2.动态库

> windows下动态库文件名一般为`xlog.dll` 。Linux下文件名一般为`libxlog.so` 。

1. 准备程序文件（如上）

2. 生成二进制文件
```
g++ -fPIC -c add.cpp -o add.o
```

3. 创建动态库

```
g++ -shared add.o -o libadd.so
```

也可以2 3步一起做
```
g++ -fPIC -shared add.cpp -o libadd.so
```

4. 使用动态库
```
//和静态库的使用相同
g++ main.cpp -L. -ladd -I main
```

## 3.第三方库

第三方库不是默认库，通常编译器默认不会链接，需要用户主动链接，比如`pthread.h`

1. 程序准备
pthread.cpp
```cpp
#include <iostream>

#include <math.h>           // 数学相关函数

#include <pthread.h>        // 线程支持库

#include <unistd.h>         // sleep 函数

  

using namespace std;

  

// 定义分支线程的线程体函数

void* task(void* arg) {

    while (1) {

        cout << "hello world" << endl;

        sleep(1);           // 休眠 1 秒

    }

    return NULL;            // 虽然不会执行到这里，但规范上需要返回

}

  

int main(int argc, const char* argv[]) {

    pthread_t tid;          // 定义线程号变量

    // 创建线程

    if (pthread_create(&tid, NULL, task, NULL) != 0) {

        cout << "pthread_create error" << endl;

        return 1;           // 创建失败，提前退出

    }

    // 主线程保持运行（防止主线程退出导致整个进程结束）

    while (1);

    return 0;

}
```

2. 编译时链接第三方库
```
g++ pthread.cpp -o build -lpthread
```
