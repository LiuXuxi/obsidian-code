# 如何关闭单词拼写的语法报错
### 方法一：完全关闭整个解决方案的拼写检查（推荐）

这是最彻底的方法，会为当前打开的所有项目关闭拼写检查。

1. 在菜单栏中，依次点击 ​**工具 (Tools)​**​ -> ​**选项 (Options)​**。
    
2. 在弹出的“选项”对话框中，在左侧的导航树中找到并展开 ​**文本编辑器 (Text Editor)​**。
    
3. 继续展开 ​**所有语言 (All Languages)​**，然后点击 ​**拼写检查 (Spell Checker)​**。
    
    - _提示：你也可以选择特定的语言（如 C#）下的“拼写检查”，来只为该语言进行设置。_
        
    
4. 在右侧的主窗口中，你会看到几个选项。取消勾选 ​**启用拼写检查 (Enable spell check)​**。
    
5. 点击 ​**确定**​ 按钮保存设置。
    

从此，所有代码文件中的拼写错误波浪线将不再显示。

---

### 方法二：仅对当前文件或特定词条禁用

如果你只是不希望某个特定文件或某些单词（如变量名、类名）显示波浪线，可以使用以下方法。

#### 1. 禁用单个文件的拼写检查

右键单击编辑器窗口的标签页（即打开的文件名），然后选择 ​**拼写检查**​ -> ​**禁用拼写检查**。

这样，拼写检查只会在这一个文件中被禁用。

#### 2. 将单词添加到词典（忽略特定词条）

如果只是某些特定单词（例如你的项目术语、变量名）被标记为错误，你可以让 VS 学会并忽略它们。

- 将鼠标悬停在有蓝色波浪线的单词上。
    
- 点击出现的 ​**螺丝刀**​ 或 ​**灯泡**​ 图标（快速操作）。
    
- 在弹出的菜单中，选择 ​**将 ‘[单词]’ 添加到词典**。
    

之后，这个单词在整个 Visual Studio 中都不会再被标记为错误。

#### 3. 使用注释指令（适用于 C#）

对于 C# 代码，你还可以使用特殊注释来指导拼写检查器。

- `// Disable spelling: [单词]`：忽略后面出现的指定单词。
    
    - 例如：`// Disable spelling: Api`会让它忽略后续的 “Api”。
        
    
- `// Enable spelling: [单词]`：重新启用对指定单词的检查。
    
- `// Ignore spelling [单词]`：与 `Disable spelling`类似。
    

---

### 方法三：通过编辑器配置 (.editorconfig) 文件

如果你的项目使用了 `.editorconfig`文件来统一代码风格，你也可以通过它来配置拼写检查。

1. 在项目根目录下打开或创建 `.editorconfig`文件。
    
2. 添加以下规则：
    
    ```
    # 完全禁用拼写检查
    [*.{cs,vb,cpp,cshtml,js,jsx,ts,tsx}]
    dotnet_diagnostic.CC0023.severity = none
    ```
    
    这条规则将拼写检查诊断（错误代号 CC0023）的严重性设置为“无”，即不显示。
    

这种方法特别适合团队项目，可以确保所有开发者的环境设置一致。

### 总结

|方法|作用范围|适用场景|
|---|---|---|
|​**方法一：全局选项**​|​**整个 Visual Studio**​|希望彻底、永久关闭所有项目的拼写检查，一劳永逸。|
|​**方法二：单个文件/词条**​|​**特定文件或单词**​|临时处理，或者只想忽略少数几个特定的术语。|
|​**方法三：编辑器配置**​|​**当前项目/解决方案**​|团队协作，希望将设置随项目代码一起共享给其他成员。|

对于大多数个人开发者，​**方法一**​ 是最简单直接的。
